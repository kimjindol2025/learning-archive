## 📦 패키지 매니저 핵심 개념

### 1. gogs.toml — 프로젝트 명세

```toml
[package]
name = "my_app"
version = "1.0.0"

[dependencies]
tokio = "1.0"
serde = "1.0"
```

**역할**:
- 프로젝트 메타데이터 정의
- 의존성 선언
- 빌드 설정
- "기록이 증명이다" 철학 구현

### 2. gogs.lock — 버전 고정

```
[[package]]
name = "tokio"
version = "1.2.3"
source = "registry+https://gogs.io"

[[package]]
name = "serde"
version = "1.0.5"
```

**역할**:
- 정확한 버전 기록
- 팀 전체 동일 환경 보장
- 재현 가능한 빌드

### 3. 의존성 해석기

```
gogs.toml (선언)
    ↓
버전 범위 계산 (tokio = "1.0" → 1.0.x 범위)
    ↓
레지스트리 조회 (gogs.io에서 가능한 버전 확인)
    ↓
최적 버전 선택 (1.0.5 선택)
    ↓
gogs.lock에 기록
    ↓
다운로드 & 캐시
```

### 4. 버전 범위 규칙

| 명세 | 범위 | 의미 |
|------|------|------|
| `1.0.0` | 정확히 1.0.0 | 이 버전만 사용 |
| `1.0` | 1.0.x | 1.0.0 이상, 2.0.0 미만 |
| `1.*` | 1.x.x | 1.0.0 이상, 2.0.0 미만 |
| `>=1.0, <2.0` | 범위 | 1.0.0 이상, 2.0.0 미만 |

**Semantic Versioning**:
- **MAJOR** (1.x.x): 호환성 깨지는 변경
- **MINOR** (x.1.x): 하위 호환 새 기능
- **PATCH** (x.x.1): 버그 수정

## 🔧 주요 명령어

### 프로젝트 초기화

```bash
$ gogs new my_project
    my_project/
    ├── gogs.toml
    ├── src/
    │   └── main.gogs
    └── Cargo.lock
```

### 의존성 추가

```bash
# 간단 추가
$ gogs add tokio
→ gogs.toml에 "tokio = \"1.0\"" 추가
→ gogs.lock 업데이트
→ 다운로드 & 캐시

# 버전 지정
$ gogs add serde@0.5.0
$ gogs add regex --version ">=0.2, <0.3"

# 선택적 의존성
$ gogs add openssl --optional

# 기능(feature) 포함
$ gogs add tokio --features "full"
```

### 빌드 및 실행

```bash
# 의존성 다운로드 (처음 한 번)
$ gogs fetch
→ gogs.lock에 기록된 정확한 버전 다운로드

# 빌드
$ gogs build
→ gogs-std 링크
→ 모든 의존성 컴파일
→ 메인 프로젝트 컴파일

# 릴리스 빌드 (최적화)
$ gogs build --release

# 직접 실행
$ gogs run
```

### 테스트 및 배포

```bash
# 테스트 실행
$ gogs test

# 문서 생성 및 보기
$ gogs doc --open

# 배포 (gogs.io에 업로드)
$ gogs publish
```

## 📚 표준 라이브러리 (gogs-std)

### 핵심 모듈

```gogs
// I/O 모듈
use gogs::io::{File, Read, Write};
use gogs::fs;

// 컬렉션
use gogs::collections::{Vec, HashMap, BTreeMap};

// 문자열
use gogs::string::{String, str};

// 수학
use gogs::math::{sqrt, sin, cos};

// 시간
use gogs::time::{Duration, Instant};

// 네트워크
use gogs::net::{TcpListener, TcpStream};
```

### 계층적 설계

```
gogs-std (최상위)
├── gogs::io (파일, 스트림)
├── gogs::fs (파일 시스템)
├── gogs::net (네트워킹)
├── gogs::collections (자료구조)
├── gogs::string (문자열)
├── gogs::math (수학)
├── gogs::time (시간)
├── gogs::error (에러 처리)
└── gogs::prelude (자주 쓰는 것들)
```

### 예제: 파일 읽기

```gogs
use gogs::io::File;
use gogs::error::Result;

fn main() -> Result<()> {
    let mut file = File::open("data.txt")?;
    let contents = file.read_string()?;
    println!("{}", contents);
    Ok(())
}
```

///
/// # 예제
/// ```gogs
/// let contents = read_file("data.txt")?;
/// println!("{}", contents);
/// ```
fn read_file(path: &str) -> Result<String> {
    // ...
}
```

**실행**:
```bash
$ gogs doc
```

**생성물**:
```
docs/
├── index.html           # 패키지 홈페이지

## 🎓 설계자의 관점: "사회적 시스템"

### 이제 당신이 고민해야 할 것들

#### 1. **접근성 (Accessibility)**
❓ 새로운 개발자가 이 언어를 쉽게 배울 수 있는가?
- 명확한 튜토리얼
- 초보자 친화적 에러 메시지
- 풍부한 예제

#### 2. **신뢰성 (Reliability)**
❓ 내가 다운로드한 패키지가 정말 안전한가?
- 패키지 서명 검증
- 보안 감시
- 악성 코드 감지

#### 3. **호환성 (Compatibility)**
❓ 2년 후에도 지금의 코드가 작동하는가?
- Semantic Versioning 엄격히 따르기
- 하위 호환성 유지
- gogs.lock 파일 자산으로 취급

#### 4. **발견 가능성 (Discoverability)**
❓ 좋은 라이브러리가 있다는 걸 사람들이 알 수 있는가?
- 좋은 검색 기능
- 카테고리 분류
- 별점 시스템

#### 5. **커뮤니티 (Community)**
❓ 개발자들이 협력할 수 있는 환경인가?
- 오픈소스 지원
- 이슈 트래킹
- 기여 가이드

## 💡 "기록이 증명이다 gogs" 실현

### v17.0에서 구현되는 철학

```
┌─────────────────────────────────────┐
│     gogs.toml (프로젝트 명세)        │
│                                     │
│  [package]                          │
│  name = "my_app"                    │
│  version = "1.0.0"                  │
│                                     │
│  [dependencies]                     │
│  tokio = "1.2.3" ← 정확한 버전      │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│     gogs.lock (버전 고정)            │
│                                     │
│  [[package]]                        │
│  name = "tokio"                     │
│  version = "1.2.3"                  │
│  source = "registry+https://gogs.io"│
│  hash = "sha256:abc123..."          │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│   Git Repository (기록 보존)         │
│                                     │
│  gogs.lock을 커밋 → 히스토리 남음    │
│  누가, 언제, 어떤 버전을 사용했는지  │
│  완벽한 재현 가능                   │
└─────────────────────────────────────┘
```

### 결과

✅ **명확함**: 모든 의존성이 명시적으로 기록됨
✅ **신뢰성**: 패키지 서명으로 위조 불가능
✅ **추적성**: 어떤 버전을 언제 사용했는지 알 수 있음
✅ **재현성**: 누구나 같은 환경에서 빌드 가능
